# GIF 动效交互拆解标准

当用户给 GIF、短视频、录屏片段，要求拆“这种交互动画怎么实现”、输出“能喂给 AI 一比一还原的提示词”时，按这套流程走。

## 先定证据边界

先把源文件信息查清楚，再写分析：

- 文件路径、来源、尺寸、帧数、总时长、采样帧。
- 这段动效在完整用户路径里的位置：触发前、触发中、触发后。
- 哪些元素是产品原始 UI，哪些是案例标注层、打码层、后期红框。
- 哪些判断来自画面证据，哪些是 `[推断]`。

不能把标注红框、讲解遮罩、打码块当成原产品设计。不能只看一张截图就写“动效实现”，必须看连续帧。

## 抽帧要求

普通文档正文要放静态 PNG 关键帧或关键帧拼图，GIF 只能作为补充。抽帧至少覆盖：

1. 初始静止态。
2. 动效开始态。
3. 主体元素入场态。
4. 峰值态，也就是光效、金额、弹窗最强的一帧。
5. 等待用户操作态。
6. 用户点击或系统收束态。
7. 动效结束后的状态变化。

如果只有 GIF 文件，优先用脚本：

```powershell
python C:\Users\zhoua\.codex\skills\case-teardown-standard\scripts\build_gif_keyframes.py "input.gif" --output-dir "outputs\gif-frames" --name "interaction" --frames 9 --make-strip
```

如果是录屏视频，用：

```powershell
python C:\Users\zhoua\.codex\skills\case-teardown-standard\scripts\build_interaction_keyframes.py "input.mp4" --start 75.2 --duration 3.5 --output-dir "outputs\interaction" --name "reward" --frames 6 --make-strip
```

需要做案例讲解红框时可以加 `--highlight x,y,w,h`。红框只属于讲解证据，不属于原始产品 UI。

## 动效拆解结构

每个关键动效都按 7 个维度写：

### 1. 用户触发

写清楚动效由什么触发：

- 用户点击。
- 抽奖结束。
- 金额变化。
- 进度满格。
- 好友助力成功。
- 系统自动提示。

### 2. 页面层级

把画面拆成层，不要只说“弹出一个红包”：

- 底层页面：原页面是否继续保留，是否变暗。
- 遮罩层：透明度、颜色、是否拦截点击。
- 主视觉层：红包、转盘、礼物、卡片、弹窗。
- 光效层：外发光、放射光、星星、烟花、金币。
- 文案层：标题、金额、说明、按钮。
- 状态反馈层：金额入账、进度更新、按钮变化、toast。

### 3. 动画时序

按秒拆，而不是泛泛写“先出现再消失”：

- `0.00s - 0.30s`：初始页面稳定。
- `0.30s - 0.60s`：遮罩和标题淡入。
- `0.60s - 1.00s`：主体弹出或放大。
- `1.00s - 2.40s`：高光停留，等待用户理解。
- `2.40s - 2.80s`：按钮或下一步出现。
- `点击后`：主体收束，数值或进度更新。

时长不确定时，写“约”。证据不足要标 `[推断]`。

### 4. 运动参数

把动效还原到可实现的参数：

- 入场：`opacity`、`scale`、`translateY`、`rotate`。
- 回弹：是否先超过目标尺寸，再回到正常尺寸。
- 光效：透明度、模糊、呼吸缩放、循环速度。
- 粒子：数量、方向、颜色、消失方式。
- 数字：是否跳动、滚动、飞入、闪光。
- 退场：缩小、上移、下落、淡出、飞向目标区域。

### 5. 状态机

把动效写成状态，而不是写成一段动画：

- `idle`：页面静止。
- `triggered`：触发奖励或提示。
- `popup-entering`：弹层进入。
- `waiting-action`：等待用户点击或系统继续。
- `collecting`：领取、收束、飞向账户。
- `settled`：金额、进度、按钮状态完成更新。

这能帮助后续 AI 或前端把效果做成交互，而不是只做死动画。

### 6. 增长目的

必须解释它为什么这么做：

- 小金额用大光效包装成大奖。
- 暗色遮罩把注意力从复杂页面拉到奖励。
- 大数字和外发光让用户确认“我真的得到了”。
- 点击收下让用户主动确认，增加投入。
- 入账动画证明奖励进账户，降低怀疑。
- 数值差距变小，推动继续完成下一步。

### 7. 风险和复用

写清楚这套动效复用时要注意：

- 奖励金额小但光效过大，可能带来误导感。
- 倒计时、提现门槛、剩余差额要和真实规则一致。
- 如果是现金类活动，要注意提现条件、到账口径和合规提示。
- 复用到非现金场景时，把“现金入账”改成“权益解锁/进度推进/礼品锁定”。

## 一比一还原提示词结构

提示词必须分成这些块，不能只写一段审美描述：

1. **目标**：生成什么格式、什么尺寸、什么场景、是否交互。
2. **基础页面**：背景、顶部栏、主卡片、按钮、底部区域。
3. **原始数值**：金额、进度、倒计时、待获得差额。
4. **动效层级**：底层、遮罩、主体、粒子、文案、按钮、反馈。
5. **时序**：每段几秒、谁先进、谁后退。
6. **运动参数**：scale、opacity、translate、easing、循环。
7. **点击后状态**：数值怎么变、页面怎么收束、是否有二次粒子。
8. **技术要求**：HTML/CSS/JS、Canvas、SVG、Lottie、状态机命名。
9. **不要做**：不要加案例红框、不要改风格、不要变成通用抽奖页。
10. **可选讲解层**：如果用户要案例图，再加红框和标注，不要默认加。

## 完成标准

交付前逐项检查：

- 已经抽帧，不是只看单张截图。
- 已写明源文件尺寸、帧数、时长。
- 已区分产品原始 UI 和讲解标注层。
- 已把动效拆成层级、时序、运动参数、状态机。
- 已写点击后或动效结束后的真实状态变化。
- 已解释增长目的和用户心理作用。
- 已输出完整提示词，可以直接交给 AI 生成 H5 / 动画 / 原型。
- 在线文档正文已放静态关键帧或拼图，没有只放 GIF 路径。
